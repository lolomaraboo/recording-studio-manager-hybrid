<tasks>
### Task 1: Manual testing session with Chrome (systematic page-by-page)

**Action:** Execute validation tests for all 45 pages following the protocol.

**Testing approach:**

1. **Setup Phase (10 minutes)**
   - Open test matrix CSV
   - Open testing protocol document
   - Login to production site with test admin account
   - Open browser DevTools Console (monitor for errors)
   - Prepare bug documentation file

2. **Admin Dashboard Testing (3-4 hours)**

   **For each of the 37 admin pages:**
   - Navigate to page via UI (not direct URL)
   - Execute validation checklist from protocol:
     - Page loads without Console errors
     - Navigation elements work (breadcrumbs, sidebar)
     - All buttons/links clickable
     - Forms validate correctly (if applicable)
     - Data displays or shows empty state
     - Responsive test (resize browser: mobile, tablet, desktop)
     - Performance check (< 3s load time)
   - Document any bugs using bug template
   - Update test matrix: Status = Pass/Fail
   - Take screenshot if bug found

   **Page order (by feature area):**
   - Dashboard home
   - Clients (list, create, edit, detail, import)
   - Sessions (list, create, edit, detail, calendar)
   - Invoices (list, create, edit, detail, preview)
   - Equipment (list, create, edit, detail)
   - Rooms (list, create, edit)
   - Projects (list, create, edit, detail)
   - Analytics (overview, revenue, sessions, clients)
   - Settings (profile, organization, billing, team, integrations)
   - Search results
   - Notifications

3. **Client Portal Testing (30-45 minutes)**

   **Logout and login as test client account**

   **For each of the 5 client portal pages:**
   - Navigate to page
   - Execute validation checklist
   - Document bugs
   - Update test matrix
   - Screenshot if needed

   **Pages:**
   - Client dashboard
   - My sessions
   - My invoices
   - My profile
   - Support/contact

4. **Super Admin Testing (30 minutes)**

   **Logout and login as super admin account**

   **For each of the 3 super admin pages:**
   - Navigate to page
   - Execute validation checklist
   - Document bugs
   - Update test matrix
   - Screenshot if needed

   **Pages:**
   - Organizations list
   - System settings
   - Audit logs

5. **Bug Documentation (continuous)**

   **For each bug discovered:**
   - Assign Bug ID (increment: BUG-001, BUG-002, etc.)
   - Fill bug template:
     - Page URL/route
     - Severity (P0/P1/P2/P3)
     - Category (UI/UX/Functional/Performance)
     - Description
     - Steps to reproduce
     - Expected vs Actual behavior
     - Screenshot (save to `.planning/phases/3.13-validation-ui-complete-toutes-pages/screenshots/`)
     - Browser version
   - Add to bugs list file: `.planning/phases/3.13-validation-ui-complete-toutes-pages/bugs-found.md`

6. **Progress Tracking**

   - Update test matrix CSV after each page tested
   - Mark Status: "Pass" (no issues) or "Fail" (bugs found)
   - Add Notes column entries for quick reference
   - Save matrix frequently (every 10 pages)
